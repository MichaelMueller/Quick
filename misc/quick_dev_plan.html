Introduction

Quick was meant to give a basic self explanatory and very abstract PHP web application framework. Usually other frameworks tend to be oversized.

The main idea for developers is to create data structure definitions. Out of the box, Quick should be able to create basic data CRUD operations (with web page scaffolding) and authorization features for clients while being extendable at any point.

Version planning

Version 1.0

Overview

Version 1.0 will provide a typical Database application with CRUD from scratch.
As a developer, if I am able to create minimal configuration (routing, database info, superuser) and to describe data objects and their relations to each other, the system should automatically provide:

  1. A way with a form to enter and store data persistently
  2. A way with a form to update data persistently
  3. A way or workflow to delete data persistently
  4. A way to search data using simple text entry
  5. A way to login
  6. A way to logout

Development plan

The system will consist of a central App class, an Arguments class, multiple AppFunction, MetaObject, Expression classes. The classes will have the following behavior:

  1. The App object: Routing, Error Handling, data object access and basic config. It gets the Arguments object in a run() method
  2. The Arguments object: Encapsulates input data from webserver env, command line or programmatically set
  3. AppFunction objects: The main logic, takes the App object (and therefore with access to the Arguments). Can create a workflow taking "routes"
  4. MetaObjects provide info about the data (storage type, conditional expressions, ownership)
  5. Expression classes are objects to restrict storage types in their value
